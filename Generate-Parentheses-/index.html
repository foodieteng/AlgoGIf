Algorithm Overview

Core Rules

At each step we have two choices:
1. Add '(' if open < n
2. Add ')' if close < open
When open == n && close == n, we found a valid result.

Why Backtracking?

We build the string one character at a time. By checking the constraints before branching, we prune all invalid prefixes early — no wasted work. Every leaf in the tree is a valid answer.

class Solution {
public:
    vector<string> generateParenthesis(int n) {
        vector<string> ans;
        dfs(ans, "", 0, 0, n);
        return ans;
    }

    void dfs(vector<string>& ans, string path,
            int open, int close, int n) {
        // Base case: found a valid combination
        if (open == n && close == n) {
            ans.push_back(path);
            return;
        }

        // Branch 1: add '('
        if (open < n) {
            dfs(ans, path + '(', open + 1, close, n);
        }

        // Branch 2: add ')'
        if (close < open) {
            dfs(ans, path + ')', open, close + 1, n);
        }
    }
};
